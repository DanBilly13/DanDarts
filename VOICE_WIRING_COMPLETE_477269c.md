## Summary

Successfully implemented defensive voice WebRTC wiring on stable remote games base (commit 477269c). Voice integration is now complete with non-blocking failure handling and idempotent session management.

## Changes Made

### 1. VoiceChatService.swift

**Updated `startSession()` method:**
- Changed signature to accept full context: `matchId`, `currentUserId`, `otherPlayerId`, `isReceiver`
- Added same-match idempotency guard (returns early if session already exists)
- Added cross-match safety guard (tears down stale session first)
- Implemented complete WebRTC bootstrap sequence:
  1. Configure audio session (early, before peer connection)
  2. Initialize peer connection factory
  3. Setup signalling channel
  4. Create peer connection
  5. Add local audio track
  6. Begin negotiation (receiver creates offer, challenger waits)
- Wrapped all bootstrap steps in defensive try-catch
- On failure: cleanup partial resources, set state to `.unavailable`, do NOT throw
- Session starts in `.connecting` state (truthful, not `.idle`)

**Added `cleanupFailedBootstrap()` helper:**
- Best-effort cleanup of partial voice resources
- Closes peer connection
- Tears down signalling channel
- Deactivates audio session
- Never throws exceptions

**Hardened `endSession()` method:**
- Marks session as `.ended` immediately
- Best-effort teardown (never throws, never blocks)
- Closes peer connection
- Sends disconnect signal (ignores failures)
- Tears down signalling channel
- Deactivates audio session
- Clears internal state
- Safe to call in any state (partial startup, full connection, already ended)

### 2. RemoteLobbyView.swift

**Updated voice session startup:**
- Determines role: `isReceiver = (match.receiverId == currentUser.id)`
- Determines other player: `otherPlayerId = isReceiver ? challengerId : receiverId`
- Passes full context to `startSession()`:
  - `matchId`
  - `currentUserId`
  - `otherPlayerId`
  - `isReceiver`
- Voice failure is explicitly non-blocking (logged but doesn't prevent match)

### 3. RemoteGameplayView.swift

**Updated voice session validation:**
- If session mismatch detected, restarts with full context
- Determines role: `isReceiver = (receiver.id == currentUserId)`
- Determines other player: `otherPlayerId = isReceiver ? challenger.id : receiver.id`
- Passes same full context as lobby
- Maintains non-blocking failure handling

## Implementation Principles Applied

### ✅ Defensive Programming
- All voice operations wrapped in try-catch
- Failures degrade gracefully to `.unavailable`
- Voice never blocks remote match flow
- Cleanup is best-effort and safe

### ✅ Idempotency
- Same-match calls return early (no duplicate sessions)
- Cross-match calls tear down stale session first
- Session validation prevents race conditions

### ✅ State Honesty
- Session starts in `.connecting` (not `.idle`)
- Only shows `.connected` when truly connected
- Shows `.unavailable` on any failure
- UI reflects actual voice state

### ✅ Non-Blocking Failures
- Audio session failure → cleanup and degrade
- Peer connection failure → cleanup and degrade
- Signalling failure → cleanup and degrade
- Match continues playable in all cases

### ✅ Voice as Attached Subsystem
- Voice observes match state
- Voice never drives navigation
- Voice never modifies match status
- Remote match flow remains authoritative

## Verified Negotiation Contract

**Receiver (accepts challenge, enters lobby first):**
1. Creates WebRTC offer
2. Sends offer via signalling channel
3. Waits for answer
4. Exchanges ICE candidates
5. Connection established

**Challenger (joins match, enters lobby second):**
1. Waits for offer
2. Receives offer via signalling channel
3. Creates WebRTC answer
4. Sends answer
5. Exchanges ICE candidates
6. Connection established

**Session ID:** Generated by receiver with offer, canonical for entire handshake

## Expected Behavior

### Voice States

**`.connecting`** - Shown during:
- Bootstrap
- Signalling negotiation
- ICE gathering
- Connection establishment

**`.connected`** - Shown when:
- Peer connection established
- ICE connection successful
- Media flowing

**`.unavailable`** - Shown when:
- Bootstrap fails
- Audio session fails
- Peer connection fails
- Signalling fails
- Permission denied

**`.failed`** - Internal state for:
- Bootstrap exception caught
